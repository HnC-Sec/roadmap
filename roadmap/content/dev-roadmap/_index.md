## Introduction
Becoming a developer is more than just learning how to write code in one language or another, especially in the modern era of AI-enabled workflows, where AI based agents can often write basic code almost as well as a human, but in a fraction of the time. In addition to being able to write code, developers need to learn and understand common design patterns, systems architecture, and how to avoid some of the most common security vulnerabilities. 

Another set of skills that sets a good developer apart from the rest of the crowd is an understanding of many of the most common tools and processes such as `git` for source control, understanding the concept of CI/CD pipelines, or understanding how to use a debugger to troubleshoot code issues. This roadmap aims to provide basic resources to get started in an array of common programming languages, as well as specfic sections for the most common non-programming skills a developer should have.
