## Applied Security

The sections in this part of the roadmap build on the foundations covered in the Basics sections. Where the Basics track focuses on *understanding how systems work*, the Applied Security track focuses on *understanding how systems break*, and how to do something constructive with that knowledge.

The sections here are designed to be followed roughly in order, as each builds on the last:

---

### Learning Path

```
Basics (Linux, Networking, Windows, Hardware, Research)
    ↓
CTF & Lab Practice          ← Start here: controlled environments, core tooling
    ↓
Penetration Testing         ← Methodology, professional engagements, reporting
    ↓
Web Application Security    ← The dominant real-world attack surface
    ↓
Bug Bounty Hunting          ← Real targets, real impact, real programmes
    ↓
Vulnerability Disclosure    ← The ethical and legal framework for all of the above
```

These are not rigid gates, and you will naturally move back and forth between sections, but if you find yourself lost in Bug Bounty concepts, it is likely worth reinforcing your Web Application Security fundamentals first.

---

### A Note on Ethics and Legality

Every section in this track assumes you are operating ethically and within the law. This means:

- **Authorisation**: Only test systems you own, have explicit written permission to test, or that fall within a clearly defined bug bounty or VDP scope.
- **Scope discipline**: Staying within defined scope is not just a programme rule — it is the line between ethical security research and a criminal offence in most jurisdictions.
- **Responsible disclosure**: If you find a vulnerability outside of an authorised engagement, the Vulnerability Disclosure section covers the correct path forward.
- **Documentation**: Keep records of your authorisation, scope, findings, and communications.
- **Hacking Is Not A Crime**: It's commonly misunderstood that hacking and cybercrime are the same, but they're not. The following resource is a good place to learn more about that difference: https://www.hackingisnotacrime.org/

If you are ever uncertain whether something is legal or authorised, stop and find out before proceeding. The security community's ability to do this work depends on every researcher operating responsibly.
